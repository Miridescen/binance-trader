# 交易策略文档

## 一、策略概述

币安合约逆势策略：做空 24h 涨幅榜 TOP 币种，赚取过度拉升后的回落收益。

**核心逻辑**：涨幅越大的币，短期回落概率越高 → 做空获利。

## 二、开仓策略

### 时间
- **限价平仓**：每天 08:30 开始挂限价单平仓（省手续费）
- **市价兜底**：每天 08:50 未成交的改市价清仓
- **开仓时间**：每天 09:05（平仓完成后开新仓）

### 标的筛选
1. 获取所有 USDT 合约 24h 行情
2. 过滤 24h 交易量 < 1000 万 USDT 的币种
3. 按 24h 涨幅降序排列
4. 从候选池（TOP_N × 6 = 60 个）中筛选：
   - **有市值**：通过 CoinGecko 验证，无市值币排除
   - **涨幅 >= 5%**：低于此阈值的不开仓
5. 取前 **10 个**币种开空单

### 开仓参数
| 参数 | 值 |
|------|-----|
| 方向 | 只开空单（多单已转模拟盘观察） |
| 数量 | 10 个币种 |
| 杠杆 | 3 倍 |
| 保证金 | 30 USDT / 单 |
| 名义价值 | 90 USDT / 单 |
| 总保证金 | 300 USDT |

### 下单方式
1. 以标记价挂 GTC 限价单
2. 每 60 秒检查未成交订单，按最新标记价换价重挂
3. 最多重试 10 次（约 10 分钟）
4. 仍未成交则改用市价单兜底

## 三、平仓策略

### 1. 动态止盈（仅空单）

根据持仓时间采用不同的 ROE 阈值，每 60 秒检查一次：

| 时间段 | 止盈阈值 | 说明 |
|--------|---------|------|
| 09:00 ~ 14:59 | ROE >= 50% | 前期波动大，高阈值让利润跑 |
| 15:00 ~ 08:59（次日） | ROE >= 20% | 后期 ROE 见顶回落，低阈值锁定利润 |

**为什么 15:00 切换？**
- 历史数据分析显示，空单平均 ROE 在开仓后 4-7 小时（约 13:00-15:00）达到峰值
- 15:00 后 ROE 开始震荡回落，继续持有的边际收益递减
- 避开 16:00 资金费率结算时间（00:00/08:00/16:00），防止结算波动干扰止盈判断
- 降低阈值可以在回撤前及时平仓，避免利润回吐

### 2. 止损（所有仓位）

| 条件 | 动作 |
|------|------|
| ROE <= -80% | 立即市价平仓 |

对应 3 倍杠杆下价格反向移动约 27%。

### 3. 定时平仓（两阶段）

| 时间 | 动作 | 手续费 |
|------|------|--------|
| 08:30 | 限价平仓（按标记价挂单，每 60s 换价） | 低（Maker） |
| 08:50 | 未成交的改市价兜底 | 高（Taker） |

限价平仓可以节省手续费（Maker 费率通常比 Taker 低 50%-70%）。

### 平仓优先级

```
止损（ROE <= -80%）> 止盈（动态阈值）> 定时平仓（08:50）
```

### 平仓原因标记

每次平仓会在 open_log 中记录 `close_reason`：

| close_reason | 说明 |
|-------------|------|
| 止盈 | 触发动态止盈阈值 |
| 止损 | ROE <= -80% |
| 定时平仓 | 每天 08:50 定时清仓 |

## 四、模拟盘

同时运行两组模拟盘用于策略对比：

### 对照组（不过滤市值）
- 涨幅榜 TOP10 空单 + 跌幅榜 TOP10 多单
- 不过滤市值，不设涨跌幅门槛
- 用途：验证市值过滤的效果

### 多单模拟组
- 与实盘相同参数：跌幅 >= 8% + 市值过滤 + TOP10
- side 标记为「模拟多」
- 用途：观察多单策略表现，评估是否值得恢复实盘多单

## 五、监控与数据

### 持仓监控（每 60 秒）
- 检查止盈/止损条件
- 每小时整点记录持仓快照（汇总 + 明细）
- 特殊快照：08:50（平仓前）、09:30（开仓后）

### 数据存储
- SQLite 数据库（trader.db）
- 6 张表：open_log、batch_summary、events_log、positions_log、positions_detail、virtual_log

### 前端展示
- http://服务器IP:8080
- 四个页面：开仓记录、持仓监控、持仓明细、模拟盘

## 六、关键文件

| 文件 | 说明 |
|------|------|
| `open_top_shorts.py` | 每日定时开仓/平仓策略 |
| `monitor_positions.py` | 持仓监控 + 止盈止损 |
| `virtual_trade.py` | 模拟盘（对照组 + 多单模拟） |
| `api.py` | Flask API 服务 |
| `db.py` | SQLite 数据库模块 |
| `binance_client.py` | 币安 API 客户端 |

## 七、服务管理

```bash
# 启动所有服务
bash start_service.sh

# 单独重启
systemctl restart binance-strategy    # 开仓策略
systemctl restart binance-monitor     # 持仓监控
systemctl restart binance-virtual     # 模拟盘
systemctl restart binance-api         # API 服务

# 查看日志
tail -f logs/binance-strategy.log
tail -f logs/binance-monitor.log
```
